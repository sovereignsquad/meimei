# Issue Ready-Gate Checklist

Use this checklist before moving an implementation issue to `Ready (NEXT)`.

Issue id:  
Title:  
Owner:  
Date:

## A. Clarity

- [ ] Objective is concrete and outcome-focused.
- [ ] Problem statement explains current gap.
- [ ] Goal states what success looks like.

## B. Scope

- [ ] In-scope bullets are explicit.
- [ ] Non-goals prevent adjacent overlap.
- [ ] Dependencies are linked and current.

## C. Implementation Contract

- [ ] Execution prompt is actionable for engineers/agents.
- [ ] Implementation expectations define required outputs.
- [ ] Acceptance checks are testable and observable.

## D. Risk and Safety

- [ ] Constraints include safety/governance boundaries.
- [ ] Risks are listed with realistic failure modes.
- [ ] Escalation condition is clear when blocked.

## E. Delivery and Review

- [ ] Delivery artifact list is specific.
- [ ] Developer notes identify likely touched surfaces.
- [ ] Review evidence format is clear (commit, files, validation).

## Ready Decision

- [ ] PASS: issue can move to `Ready (NEXT)`.
- [ ] FAIL: issue remains in backlog until fixed.
