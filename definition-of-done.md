# Definition of Done

A change is done only when all of the following are true:

- the intent is clear
- the scope is bounded
- the relevant docs are updated
- the skill catalog is consistent
- the result can be explained to OC in plain language
- there is no obvious duplication or contradiction

For higher-risk work, also require:

- explicit OC review
- security check
- verification notes

For new product functions, also require:

- the function delivery lifecycle was followed
- the route is stable and documented
- the back path to the dashboard root exists
- the primary action and output are obvious in the UI
